## 0. 목적과 경계

사용자 목표: "더 플랫폼답게 제품화" — 특히 **온보딩/셀프서브 경험**(첫 진입, 빈 상태, 커맨드팔레트, "처음 써도 헤매지 않는" 감각). wave-037/059/060과 동일한 전략: 공용 프리미티브/셸 레벨에서 구현해 전체 화면에 자연스럽게 적용되게 하고, 라우트/API/기능은 변경하지 않는다.

**리서치에서 반박된 것(채택 안 함)**: "체크리스트 완료율 X%", "투어 이탈률 Y%", "80%가 첫 주에 이탈" 등 모든 정량 수치(전부 0-3 반박 — 근거 없는 벤더 마케팅 수치). **채택하는 것은 정성적 패턴만**: empty state 공식, 체크리스트 항목 수 범위, 커맨드팔레트 3변형, 샌드박스/샘플 패턴.

## 2. 구현 범위 (additive, 라우트/API 불변)

1. **커맨드팔레트** (신규, 가장 큰 항목): `Cmd/Ctrl+K`로 열리는 모달. `navigation.ts`의 57개 라우트를 네비게이션 액션으로, "새 프로젝트 만들기"를 생성 액션으로, 최근 프로젝트(이미 있는 `recentProjectStorageKey` localStorage 값)를 우선 노출. 퍼지 검색, 키보드 상하 이동+Enter, Esc로 닫기, 포커스 트랩, 접근성(role=dialog, aria-label). 상단바에 발견 가능한 버튼(예: "검색... ⌘K")도 함께 배치.
2. **Empty state 강화**: `PageState`(이미 wave-059에서 size variant 보유)의 empty variant에 아이콘 슬롯 + 단일 주 CTA + 선택적 보조 CTA 지원 추가(옵트인, 기존 20+ 호출부 100% 회귀 없음). Dashboard/ProjectListPage의 "프로젝트 0개" 상태에 P2 적용.
3. **프로젝트 홈 온보딩 체크리스트**: 신규 위젯(또는 프로젝트 홈에 조건부 카드). 항목: ① 온톨로지 클래스/관계 1개 이상 정의됨 ② 소스 1개 이상 업로드됨 ③ 추출 잡 1개 이상 실행됨 ④ 게시 그래프 버전 1개 이상 존재. 각 항목은 **이미 API로 제공되는 카운트**(프로젝트 홈이 이미 표시 중인 Sources/Ontology Versions 등)로 완료 판정 — 새 상태 저장·백엔드 변경 없음. 전부 완료되면 카드 자동 숨김(닫기 버튼도 제공, localStorage로 숨김 여부 기억).

## 3. 범위 밖 (이번 wave 제외)

- **툴팁 투어(product tour)**: 각 화면에 핀포인트 툴팁을 다는 것은 페이지별 계측이 많이 필요해 리스크 대비 효용이 낮음 — 체크리스트+개선된 empty state로 우선 대체, 향후 후보로 기록.
- **백엔드 샘플 데이터 시딩 기능**: "샘플로 시작"은 **이미 존재하는 프로젝트로 이동**만 하며, 새로 시딩하는 백엔드 기능은 만들지 않는다(mock 모드에서는 기존 데모 프로젝트로 자연스럽게 연결됨; actual API 모드에서 프로젝트가 0개면 보조 CTA 자체를 숨긴다).
- 인앱 헬프 사이드바: 범위 밖, 후속 후보.

## 4. 완료 기준

- `npm run build`/`test` PASS, `git diff --check` clean.
- Cmd/Ctrl+K로 팔레트가 열리고, 57개 라우트 중 아무 라우트나 검색→Enter로 이동 가능. Esc로 닫힘. 상단바에 발견 가능한 트리거 버튼 존재.
- 기존 empty state 20+ 호출부 시각적 회귀 없음(옵트인 확인).
- 프로젝트 홈 체크리스트가 실제 카운트 기반으로 정확히 완료 표시, 전부 완료 시 자동 숨김 + 수동 닫기 가능.
- 모바일 드로어/데스크톱 사이드바 collapse(wave-058/059) 회귀 없음.
