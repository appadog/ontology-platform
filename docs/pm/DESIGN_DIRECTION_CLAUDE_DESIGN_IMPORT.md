## 0. What this file is

A self-contained interactive prototype (`.dc.html`, a Claude-Design "design comp" format with its own tiny `sc-for`/`sc-if` templating + `DCLogic` state class) mocking up **5 real screens of THIS product** with hardcoded demo data: Dashboard, Projects, Ontology (modeler), Sources, Review (inbox). It is **not a new product** — it uses our exact current dashboard copy ("문서에서 추출한 지식을... 근거를 남기고 검수한 뒤 게시합니다", "후보와 게시를 분리합니다" / "모든 항목에 근거가 남습니다" / "품질을 지속적으로 추적합니다", the same 4-metric grid Active Projects/Sources/Ontology Classes/Relations), the same sidebar width (248px), and the same nav grouping (BUILD/REVIEW) as our real `AppShell`. It is a **visual redesign proposal for our existing screens**, to be applied with our real data/routes/components — not a rebuild.

**This is presentation-only.** No new routes, no new API calls, no functionality changes. Every real nav item, tenant switcher, command palette, etc. must keep working — the mock only shows a demo SUBSET of nav items (Dashboard/Projects/Admin/Ontology/Sources/Review); apply the new visual treatment to the FULL real nav (all ~18 items across BUILD/REVIEW/PUBLISH/ANALYZE), do not truncate it to match the mock.

## 3. Constraints (do not violate)

- **Additive/value-only token changes** — no renamed or removed theme keys; every existing consumer must keep compiling and rendering sensibly with the new color VALUES.
- **No functionality changes** — every real route, nav item, API call, mutation, filter, and existing interactive behavior (wave-058 F1-F6, wave-059 sidebar collapse, wave-060/061 CompactTable, wave-063 command palette/empty-state/onboarding checklist) must keep working exactly as before. This wave is presentation-only on top of all of that.
- **Reuse real components** — `StatusBadge`, `HanaBadge`, `HanaButton`, `HanaCard`, `CompactTable`, `PageState`, `PageContainer` are the vocabulary; do not fork parallel one-off styled components that duplicate what these already do. Where the mock's inline styles imply a new small treatment (e.g. the avatar-initials chip, the dark hero), it's fine to add a small new styled-component, but check first whether an existing primitive already covers it.
- **Self-hosted fonts only** (no Google Fonts CDN link — the mock's `<link href="https://fonts.googleapis.com/...">` must NOT be copied verbatim; use `@fontsource/newsreader` self-hosted, same reasoning as wave-059's Inter choice, documented in `docs/DEPLOYMENT.md`).
- **Full nav preserved** — do not truncate the LNB to the mock's 6-item demo subset. All ~18 real nav items keep their routes; only the visual treatment (hover/active/spacing/label typography) changes.
